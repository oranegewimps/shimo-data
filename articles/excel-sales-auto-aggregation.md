# Excelで売上を自動集計する3つのステップ【ピボットテーブル不要】

![](https://github.com/oranegewimps/shimo-data/blob/main/articles/images/excel-eyecatch.png?raw=true)

## こんな悩みありませんか?

- 毎月Excelで売上を手作業で集計している
- VLOOKUP関数を使ってるけど、エラーが出ると直せない
- 「もっと楽にできる方法があるはず」と思いつつ、時間がない

実は、**たった3つの関数の組み合わせ**で、売上集計は自動化できます。

私はBIコンサルとして20社以上のデータ可視化を支援してきましたが、中小企業の8割は「Excel関数を少し工夫するだけ」で劇的に改善します。ピボットテーブルやマクロを覚える必要はありません。

この記事では、誰でも再現できる売上自動集計の仕組みを、実例を交えて解説します。

## この記事で分かること

- SUMIFS関数で商品別・月別売上を自動計算する方法
- UNIQUE関数で商品リストを自動生成する仕組み
- データが増えても壊れない集計表の作り方

**所要時間**：15分程度で読めて、30分で自分のExcelに実装できます。

## ステップ1：元データを「1行1取引」の形にする

### なぜこの形が重要なのか

多くの人が集計で苦労する原因は、**元データの構造**にあります。

![](https://github.com/oranegewimps/shimo-data/blob/main/articles/images/excel-good-bad.png?raw=true)

❌ **よくある失敗例：月ごとにシートを分けている**
```
4月シート、5月シート、6月シート...
```

⭕ **推奨する形：すべて1つのシートに並べる**
```
日付       | 商品名 | 金額
2026-04-01 | 商品A | 10,000
2026-04-01 | 商品B | 15,000
2026-04-05 | 商品A | 10,000
```

この形にすると、関数1つで「いつでも・どんな切り口でも」集計できるようになります。

### 実装のポイント

**必要な列はこの3つだけ**

1. 日付列（YYYY-MM-DD形式）
2. 商品名列
3. 金額列

**注意点**

- 日付は必ず「2026-04-01」形式（スラッシュ区切りでもOK）
- 商品名の表記ゆれに注意（「商品A」と「商品 A」は別物）
- 空白行を入れない

既存のデータがバラバラでも大丈夫です。コピー&ペーストで1つのシートにまとめるだけで準備完了です。

## ステップ2：SUMIFS関数で月別・商品別に集計する

### SUMIFS関数とは

「条件に合うデータだけを合計する」関数です。VLOOKUPより柔軟で、エラーも出にくい特徴があります。

![](https://github.com/oranegewimps/shimo-data/blob/main/articles/images/excel-sumifs.png?raw=true)

### 基本の書き方

```excel
=SUMIFS(合計範囲, 条件範囲1, 条件1, 条件範囲2, 条件2)
```

### 実例：4月の商品A売上を計算

```excel
=SUMIFS(C:C, A:A, ">=2026-04-01", A:A, "<2026-05-01", B:B, "商品A")
```

**解説**

- `C:C`：金額列全体を合計
- `A:A, ">=2026-04-01"`：日付が4月1日以降
- `A:A, "<2026-05-01"`：日付が5月1日より前
- `B:B, "商品A"`：商品名が「商品A」

この数式で、「4月の商品A売上」が自動計算されます。

### セル参照で動的にする

毎回日付や商品名を書き換えるのは面倒なので、セル参照を使います。

```excel
=SUMIFS($C:$C, $A:$A, ">="&$E2, $A:$A, "<"&$F2, $B:$B, $D2)
```

- E2セル：開始日（2026-04-01）
- F2セル：終了日（2026-05-01）
- D2セル：商品名（商品A）

この形にすると、E2・F2・D2を変えるだけで、自動的に集計結果が変わります。

## ステップ3：UNIQUE関数で商品リストを自動生成

### なぜUNIQUE関数が必要か

商品が増えるたびに手動で行を追加するのは手間です。UNIQUE関数を使えば、**データに商品が追加されたら、自動的に集計表にも反映**されます。

### 基本の書き方

```excel
=UNIQUE(B:B)
```

これだけで、B列（商品名列）の重複を除いたリストが自動生成されます。

### 実装例

**集計表シートの構成**

| 商品名（自動生成） | 4月売上 | 5月売上 |
|---|---|---|
| =UNIQUE(元データ!B:B) | =SUMIFS(...) | =SUMIFS(...) |

D列にUNIQUE関数を配置すると、商品が増えても自動的に行が追加されます。

### 注意点

- **Excel 2021 / Microsoft 365でのみ使用可能**
- 古いバージョンの場合は、手動で商品リストを作成

古いExcelを使っている場合は、ピボットテーブルか、商品リストを手動メンテナンスする方法になります。

### シート構成

**シート1：元データ**

```
2026-04-10 | 商品B  | 15,000
```

**シート2：集計表**

### 関数の配置

- B2セル：`=SUMIFS(元データ!$C:$C, 元データ!$A:$A, ">=2026-04-01", 元データ!$A:$A, "<2026-05-01", 元データ!$B:$B, $A2)`
- A2セル：`=UNIQUE(元データ!B:B)`（または手動入力）

この形にすると、**元データに取引を追加するだけで、集計表が自動更新**されます。

## よくある質問

**Q1. データが1万行以上ある場合、重くなりませんか？**

A. SUMIFS関数は10万行程度なら問題なく動作します。ただし、UNIQUE関数を多用すると遅くなる場合があるため、商品数が100を超える場合は手動リスト推奨です。

**Q2. 日付の範囲を自動的に「今月」にする方法は？**

A. EOMONTH関数を組み合わせると、「今月1日〜今月末日」を自動計算できます。

```excel
開始日：=EOMONTH(TODAY(),-1)+1
終了日：=EOMONTH(TODAY(),0)+1
```

**Q3. VLOOKUPとの使い分けは？**

A. VLOOKUPは「表から値を引っ張る」用途、SUMIFSは「条件で集計する」用途です。集計にはSUMIFSの方が柔軟でエラーも少なくなります。

## まとめ：自動集計の3ステップ

1. **元データを1行1取引の形に整える**
2. **SUMIFS関数で条件付き集計**
3. **UNIQUE関数で商品リスト自動生成**

この仕組みを一度作れば、毎月の集計作業は「元データに貼り付けるだけ」になります。

手作業での集計に毎月2時間かけているなら、年間24時間の削減です。この記事の実装に30分かけても、2ヶ月で元が取れます。
